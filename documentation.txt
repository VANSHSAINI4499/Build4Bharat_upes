# Sparshgyan: Project Documentation

## 1. Project Overview
Sparshgyan is an accessibility-focused hackathon project that combines a learning interface with assistive interaction modes for users with visual, hearing, and motor accessibility needs.

The repository currently contains three working parts:
- A React + Vite frontend for the learning experience and accessibility controls.
- An Arduino sketch that converts typed letters into tactile vibration patterns based on Braille dot mappings.
- A Python virtual mouse module that uses hand tracking for gesture-based control.

The overall idea is to let users interact with educational content through multiple access modes: voice guidance, on-screen Braille-like subtitles, gesture navigation, and tactile output.

## 2. Repository Structure

### A. Frontend Web Application (`Gthon-Repo/client/`)
This is a React application built with Vite.

Important files:
- `src/App.jsx`: Defines the client routes and mounts the global voice assistant.
- `src/components/New.jsx`: Main landing/dashboard page that assembles the homepage sections.
- `src/components/Productdetails.jsx`: Displays a sample course/product page and embeds the Braille subtitle panel.
- `src/components/Video.jsx`: Shows a sample learning video page with related content and subtitles.
- `src/components/VisionAssist.jsx`: Dedicated vision-assist screen for hands-free interaction messaging.
- `src/components/VoiceAssistant.jsx`: Adds speech recognition, speech synthesis, and voice-based navigation commands.
- `src/components/AccessibilityControls.jsx`: Contains controls for Arduino connection via Web Serial and a local gesture toggle button.
- `src/components/Subtitles.jsx`: Animates sample text into Braille Unicode characters for visual accessibility.

Frontend routes:
- `/` and `/new`: Main dashboard
- `/product`: Product/course details page
- `/video`: Video learning page
- `/vision`: Vision Assist page

Tech stack:
- React 18
- Vite
- React Router
- Tailwind CSS + custom CSS
- Browser APIs: Speech Recognition, Speech Synthesis, Web Serial

### B. Arduino Tactile Output (`AdrinoCodes/bil369/bil369.ino`)
This sketch runs on an Arduino and maps letters `A-Z` to 6-dot Braille patterns.

How it works:
- Serial input is read at `9600` baud.
- Each valid alphabet character is converted to a predefined Braille dot array.
- A vibration motor attached to PWM pin `9` is pulsed to represent active Braille dots.
- Inactive dots are represented by timing gaps, allowing the user to feel the pattern sequentially.

This module is intended to provide physical tactile feedback for typed text.

### C. Python Gesture / Virtual Mouse (`virtual-Mouse/`)
This module provides gesture-based mouse control using computer vision.

Important files:
- `app.py`: Flask API for starting and stopping the gesture mouse script as a separate process.
- `virtualmouse.py`: Uses OpenCV, MediaPipe, PyAutoGUI, and Pynput to detect hand gestures and convert them into mouse actions.
- `util.py`: Helper math functions used for gesture calculations.
- `requirements.txt`: Python dependencies.

Supported gesture actions in the current script:
- Cursor movement
- Left click
- Right click
- Screenshot capture

Note:
- The React frontend currently only toggles a local gesture state in `AccessibilityControls.jsx`.
- It does not yet call the Flask endpoints in `virtual-Mouse/app.py`.
- That means the UI and the Python gesture service are present in the repo, but not fully wired together yet.

## 3. Feature Flow

### Voice Assistance
`VoiceAssistant.jsx` is rendered globally, so it remains available across the app.

Current supported commands include:
- `Read this page`
- `Stop reading`
- `Go to AI course`
- `Open vision assist`
- `Go to home`
- `Stop listening`

The component uses:
- Speech recognition for command input
- Speech synthesis for spoken feedback
- React Router navigation for moving between pages

### Visual Braille Output
`Subtitles.jsx` converts sample text into Braille Unicode characters and reveals them progressively on screen.

This creates a visual accessibility layer that complements the tactile Arduino module.

### Tactile Device Output
`AccessibilityControls.jsx` can connect to an Arduino through the browser Web Serial API.

When connected:
- The user enters a word or sentence.
- Non-alphabetic characters are filtered out.
- Letters are sent one by one to the Arduino.
- A `2000 ms` delay is used between letters so the hardware has time to finish vibrating each character pattern.

### Gesture Control
The repository includes a separate Python-based gesture mouse system. It is designed to support hands-free interaction, especially for users who may have difficulty with a conventional mouse.

At the moment:
- The Python gesture engine exists and can run independently.
- The frontend button changes UI state only.
- Full frontend-to-Flask integration is a future enhancement.

## 4. Current Architecture Summary
This project is currently best understood as a multi-module prototype rather than a single fully integrated production system.

Present integration status:
- Frontend and Arduino connection: partially integrated through Web Serial in the browser.
- Frontend and voice assistant: integrated in the React app.
- Frontend and Python virtual mouse: not yet integrated through API calls.
- Backend database/authentication layer: not present in the current repository.

## 5. How to Run the Project

### A. Run the React Frontend
1. Open a terminal in `Build4Bharat_upes/Gthon-Repo/client`.
2. Run `npm install`.
3. Run `npm run dev`.
4. Open the local Vite URL shown in the terminal, usually `http://localhost:5173`.

### B. Run the Arduino Module
1. Connect the Arduino board to your computer.
2. Connect the vibration motor to the Arduino circuit as required by your hardware setup.
3. Open `AdrinoCodes/bil369/bil369.ino` in the Arduino IDE.
4. Upload the sketch.
5. Open the Serial Monitor and use baud rate `9600`.
6. Type letters `A-Z` to test tactile output.

### C. Run the Python Virtual Mouse Service
1. Open a terminal in `Build4Bharat_upes/virtual-Mouse`.
2. Install dependencies with `pip install -r requirements.txt`.
3. Start the Flask API with `python app.py`.
4. The service runs on `http://localhost:5000`.

If needed, the gesture engine can also be run directly through `virtualmouse.py`.

## 6. Known Limitations
- The documentation previously referenced a backend authentication system, but that system is not part of the current repo.
- The gesture toggle in the frontend does not yet start or stop the Flask virtual mouse service.
- Some displayed course and video content is placeholder/demo content.
- Browser-based features such as microphone access and Web Serial require user permission and compatible browsers.
