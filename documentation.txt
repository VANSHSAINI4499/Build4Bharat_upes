# Sparshgyan: Project Documentation

## 1. Project Overview
Sparshgyan is an accessibility-first hackathon project designed to make digital education reachable for users with visual, hearing, and motor disabilities. It combines a Next.js web platform with a Python hand-gesture engine and an Arduino haptic Braille controller.

The repository contains three working parts:
- A Next.js 14 (App Router) frontend with TypeScript and Tailwind CSS — rebuilt from the earlier React/Vite prototype.
- An Arduino sketch that maps letters A–Z to tactile 6-dot Braille vibration patterns over serial.
- A Python virtual mouse module that uses computer vision (MediaPipe) to convert hand gestures into mouse events, exposed via a Flask API.

Users can interact with educational content through four access modes: voice-command navigation, on-screen Unicode Braille output, gesture-based cursor control, and live speech-to-text captions with real-time haptic feedback.

- `badge.tsx`, `button.tsx`, `switch.tsx` — Radix UI-based styled primitives.

#### Hooks (`lib/hooks/`)

- `useSpeech.ts` — Wraps the Web Speech Recognition API (`continuous = true`, `interimResults = true`, `en-US`). Maintains a rolling 30-word window (`windowText`) and live partial result (`interimText`). On each final result: optionally triggers TTS, triggers Auto Vibrate (calls `enqueueRef`), and fires an `onVoiceCommand` callback (used by `VoiceAssistant`). Auto-restarts on network errors up to 10 times with backoff.
- `useSerial.ts` — Wraps the Web Serial API. Connects to the Arduino at 9600 baud. `enqueue(text)`: splits text into words and pushes to an in-memory queue. `drainQueue()`: sends one character byte at a time with a **2000 ms delay** between each character matching the Arduino's vibration timing. Updates `vibrateProgress` in the store.
- `useGesture.ts` — Calls the Next.js proxy endpoints (`/api/gesture/start` and `/api/gesture/stop`) to start/stop the Python gesture service. Updates `gestureStatus` (`starting` → `running` or `error`) and `gestureActive` in the Zustand store.

#### Utilities (`lib/`)
- `braille.ts` — Pure utility. `brailleMap` lookup table for a–z, 0–9, and punctuation (`. , ! ? -`) → Unicode Braille characters. `toBraille(text)` lowercases input and maps each character.
- `utils.ts` — `cn()` helper combining `clsx` + `tailwind-merge`.

#### Global State (`store/useAppStore.ts`)
Single flat Zustand store. State domains:

| Domain | Key Fields |
|---|---|
| Captions / Speech | `windowText`, `interimText`, `isListening`, `statusMsg`, `micError` |
| Arduino / Haptics | `arduinoConnected`, `autoVibrate`, `vibrateProgress` |
| Gesture | `gestureActive`, `gestureStatus` (`idle \| starting \| running \| error`) |
| UI Preferences | `autoScroll`, `showBraille`, `ttsEnabled` |
| Accessibility Score | `accessibilityScore` (default: 72) |

### B. Arduino Tactile Output (`AdrinoCodes/bil369/bil369.ino`)
Runs on an Arduino and maps letters A–Z to 6-dot Braille patterns.

How it works:
- Serial input is read at `9600` baud.
- Each valid alphabet character is converted to a predefined Braille dot array.
- A vibration motor on PWM pin `9` is pulsed to represent active Braille dots.
- Inactive dots are represented by timing gaps so the user can feel each pattern sequentially.
- Each character takes approximately `2000 ms` to complete (6 dots × 250 ms + 500 ms gap).

Used for tactile output from two sources:
- `ArduinoPanel.tsx` — manual "Send Captions Now" or Auto Vibrate from any page.
- Directly from the `/captions` page sidebar.

### C. Python Gesture / Virtual Mouse (`virtual-Mouse/`)
Provides gesture-based hands-free cursor control using computer vision.

Important files:
- `app.py` — Flask API with `/api/start-mouse` and `/api/stop-mouse` endpoints. Spawns/terminates the gesture script as a subprocess.
- `virtualmouse.py` — Uses OpenCV, MediaPipe, PyAutoGUI, and Pynput to detect hand landmarks and translate them into cursor movement, left click, right click, and screenshot actions.
- `util.py` — Helper math functions for gesture angle/distance calculations.
- `requirements.txt` — Python dependencies.
- `venv/` — Local Python virtual environment (not committed to git).

Supported gesture actions:
- Cursor movement
- Left click
- Right click
- Screenshot capture

Integration status: **fully wired**. The Next.js frontend calls the Flask endpoints via `/api/gesture/start` and `/api/gesture/stop` proxy routes.

## 3. Feature Flow

### Voice Navigation
`VoiceAssistant.tsx` is mounted globally in the root layout and available on every page (except `/captions` which manages its own mic).

Supported commands:
- `open captions` / `open live captions` → `/captions`
- `go home` → `/`
- `open vision` / `vision assist` → `/vision`
- `open video` → `/video`
- `open course` / `open courses` → `/product`

### Live Captions (Speech-to-Text for Deaf Users)
`/captions` page powered by `useSpeech` hook.

- `continuous = true`, `interimResults = true` — recognition fires 10–20 events/second.
- Interim (partial) results show live as the user speaks.
- Final results are merged into a rolling 30-word window — older words are dropped, keeping memory flat.
- Auto Vibrate: every finalised phrase is queued and sent to the Arduino letter by letter as it arrives.
- TTS toggle reads each finalised phrase aloud via `SpeechSynthesis`.
- Auto-scroll and RAF throttling keep the UI smooth during rapid recognition.
- Network errors from Chrome's speech service are silently retried up to 10 times with backoff.

### Haptic Braille Output
`ArduinoPanel.tsx` + `useSerial.ts` + `bil369.ino` work together:
1. User connects the Arduino via Web Serial from the sidebar panel.
2. `enqueue(text)` splits words into a queue; `drainQueue()` sends one character at a time with 2000 ms spacing.
3. The Arduino vibrates the corresponding Braille dot pattern for each letter.
4. A progress indicator shows the current character (e.g. `📳 H (1/5) — "hello"`).

### Visual Braille Display
`BrailleDisplay.tsx` converts `windowText` to Unicode Braille characters via `toBraille()` in `lib/braille.ts`. Displayed in large amber monospace text. Visual reference only — no direct hardware output.

### Gesture Control
`GestureControl.tsx` + `useGesture.ts` + `app/api/gesture/*` + `virtual-Mouse/app.py`:
1. User clicks Start on the Vision Assist page.
2. `useGesture.start()` POSTs to `/api/gesture/start`.
3. The Next.js route handler proxies to `http://localhost:5000/api/start-mouse`.
4. Flask spawns `virtualmouse.py` as a subprocess.
5. MediaPipe detects hand landmarks in the webcam feed and drives the cursor.

## 5. How to Run the Project

### A. Run the Next.js Frontend
1. Open a terminal in `sparshgyan-next/`.
2. Run `npm install`.
3. Run `npm run dev`.
4. Open `http://localhost:3000`.

### B. Run the Arduino Module
1. Connect the Arduino board to your computer.
2. Wire the vibration motor to PWM pin `9`.
3. Open `AdrinoCodes/bil369/bil369.ino` in the Arduino IDE.
4. Upload the sketch (baud rate `9600`).
5. On the web app, click "Connect Arduino" in any sidebar panel to pair the device via Web Serial.

### C. Run the Python Virtual Mouse Service
1. Open a terminal in `virtual-Mouse/`.
2. Activate the virtual environment: `.\venv\Scripts\activate` (Windows) or `source venv/bin/activate` (macOS/Linux).
3. Install dependencies: `pip install -r requirements.txt`.
4. Start the Flask API: `python app.py`.
5. The service runs on `http://localhost:5000`.
6. On the web app, navigate to `/vision` and click "Start Gesture Service".

### D. Use the Live Captions Feature
1. Start the Next.js frontend.
2. Navigate to `http://localhost:3000/captions` or say "open captions" to the voice assistant.
3. Allow microphone permissions when prompted.
4. Click "Start Listening".
5. Spoken words appear as rolling captions.
6. Optionally connect the Arduino and enable "Auto Vibrate" to feel each word as it is spoken.

Note: Live Captions requires Chrome or Edge. The Web Speech API sends audio to Google's servers and requires an active internet connection.

## 6. Known Limitations
- Browser-based features (microphone, Web Serial, camera) require Chrome or Edge and explicit user permissions.
- Live Captions requires an internet connection — audio is processed by Google's speech recognition servers.
- The Visual Braille display (`BrailleDisplay`) is a Unicode reference only; hardware output requires the Arduino to be connected and Auto Vibrate enabled.
- Course and video content is currently placeholder/demo data.
- The video page embeds a static YouTube iframe; replace the video ID for actual course content.
